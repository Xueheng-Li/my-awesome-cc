# Slides 页面类型详解

## 概述

根据内容特点选择合适的页面类型，是制作高质量slides的关键。本文档详细说明每种页面类型的使用场景和格式规范。

## 一、标题页（Title Slide）

### 使用场景
- 演示文稿开场
- 新章节开始（可选）

### 格式模板
```markdown
# 主标题

**副标题说明**

**补充描述**

作者，机构，日期
```

### 内容规范
- 主标题：简洁有力，10字以内
- 副标题：说明范围或主题，15字以内
- 作者信息：1-2行

### 示例
```markdown
# AI 如何改变科学研究

**从假设生成到实验设计的全流程革新**

张三，北京大学
2024年12月
```

## 二、目录页（Agenda/Outline）

### 使用场景
- 演示开始时预告内容
- 长演示中的导航

### 格式模板
```markdown
## 今日内容

1. **第一部分** - 简要说明
2. **第二部分** - 简要说明
3. **第三部分** - 简要说明
4. **第四部分** - 简要说明
```

### 内容规范
- 3-5个主要部分
- 每项配简短说明（5-10字）
- 使用有序列表

## 三、单点页（Single Point）

### 使用场景
- 强调核心观点
- 重要结论展示
- 金句/名言引用

### 格式模板
```markdown
## 核心观点

**关键陈述用粗体强调。**

普通文本用于解释和说明。

目标：**用一句话概括本页要点。**
```

### 内容规范
- 一个核心观点
- 2-3行解释
- 视觉焦点明确

## 四、列表页（List/Bullet Points）

### 使用场景
- 多个并列要点
- 步骤流程
- 要素列举

### 格式模板（无序列表）
```markdown
## 要点标题

**核心要素：**

- **第一点** - 解释说明
- **第二点** - 解释说明
- **第三点** - 解释说明
- **第四点** - 解释说明
```

### 格式模板（有序列表）
```markdown
## 四步流程

1. **步骤一** - 说明
2. **步骤二** - 说明
3. **步骤三** - 说明
4. **步骤四** - 说明
```

### 内容规范
- 列表项：3-6个
- 每项配简短说明
- 要点使用粗体

### 密度控制
| 项数 | 状态 | 建议 |
|------|------|------|
| 1-3 | 偏少 | 可合并或扩展 |
| 4-6 | 理想 | 保持 |
| 7-8 | 偏多 | 考虑拆分 |
| 9+ | 过多 | 必须拆分 |

## 五、对比页（Comparison）

### 使用场景
- 两种方案对比
- 优缺点分析
- 前后对比
- 正误对比

### 格式模板（两栏对比）
```markdown
## 详细对比

**两种方式：**

1. **方式一**
   - 特点：描述
   - 用途：说明

2. **方式二**
   - 特点：描述
   - 用途：说明
```

### 格式模板（正误对比）
```markdown
## 对比展示：错误 vs 正确

**错误示例**
> 这是一个不好的例子，展示常见错误。

**正确示例**
> 这是一个好的例子，展示正确做法。
```

### 内容规范
- 对比项控制在2-3个
- 每项内容对等
- 使用相同格式便于比较

## 六、引用页（Quote）

### 使用场景
- 重要引语
- 专家观点
- 研究结论

### 格式模板
```markdown
## 核心规则

**"引用经典表述。"**

> 重要引用使用 blockquote 格式，会显示为蓝色带左边框。

补充解释和说明。
```

### 内容规范
- 引用内容突出显示
- 标注来源
- 配简短解释

## 七、分隔页（Section Divider）

### 使用场景
- 章节过渡
- 主题切换
- 节奏调节

### 格式模板
```markdown
# Part 2: 第二部分标题
```

### 内容规范
- 仅标题，无其他内容
- 使用h1标题
- 可加Part编号

## 八、练习/示例页（Exercise/Example）

### 使用场景
- 案例展示
- 练习题目
- 修改示例

### 格式模板
```markdown
## 练习环节

**修改前：**
> 冗长、复杂、不清晰的表述示例。

**修改后：**
> 简洁、清晰、有力的表述示例。
```

### 内容规范
- 对比明确
- 改进点突出
- 引导思考

## 九、结构图页（Structure/Framework）

### 使用场景
- 框架展示
- 流程说明
- 层次结构

### 格式模板
```markdown
## 四要素结构

1. **主题句** - 表明段落中心思想
2. **逻辑流** - 每句承接上句
3. **证据** - 数据、例子、推理支撑
4. **高潮** - 最重要的内容放最后
```

### 内容规范
- 层次清晰
- 关系明确
- 可配图辅助

## 十、总结页（Summary）

### 使用场景
- 章节结束
- 演示收尾
- 要点回顾

### 格式模板
```markdown
# 总结：三个层次

**句子层面：**
- 要点一
- 要点二

**段落层面：**
- 要点一
- 要点二

**整体层面：**
- 要点一
- 要点二
```

### 内容规范
- 回顾核心要点
- 分类整理
- 简洁有力

## 十一、结束页（Closing）

### 使用场景
- 演示结尾
- Call to Action

### 格式模板
```markdown
# 最终目标

**好的表达兼具风格与实质。**

它创造**思维节奏**，引导读者清晰、有力地理解复杂观点。

**表达思想，而非炫耀技巧。**
```

### 内容规范
- 核心信息
- 留下印象
- 可加致谢/联系方式

## 页面类型选择指南

| 内容特点 | 推荐页面类型 |
|----------|-------------|
| 开场介绍 | 标题页 |
| 内容预告 | 目录页 |
| 单个核心观点 | 单点页 |
| 多个并列要点 | 列表页 |
| 两种选择对比 | 对比页 |
| 重要引语 | 引用页 |
| 章节过渡 | 分隔页 |
| 案例分析 | 练习/示例页 |
| 框架模型 | 结构图页 |
| 内容回顾 | 总结页 |
| 演示结束 | 结束页 |

## 页面组合建议

### 标准演示结构（15-20页）
1. 标题页 (1页)
2. 目录页 (1页)
3. 第一部分：分隔页 + 内容页 (3-5页)
4. 第二部分：分隔页 + 内容页 (3-5页)
5. 第三部分：分隔页 + 内容页 (3-5页)
6. 总结页 (1页)
7. 结束页 (1页)

### 短演示结构（8-12页）
1. 标题页 (1页)
2. 核心内容 (5-8页)
3. 总结页 (1页)
4. 结束页 (1页)
