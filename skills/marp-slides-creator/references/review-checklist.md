# Slides 审阅检查清单

## 一、内容检查

### 完整性
- [ ] 核心论点是否都已覆盖？
- [ ] 逻辑链条是否完整？
- [ ] 是否有必要的背景介绍？
- [ ] 结论是否清晰明确？

### 准确性
- [ ] 数据和事实是否正确？
- [ ] 引用是否准确标注？
- [ ] 术语使用是否一致？

### 简洁性
- [ ] 是否有冗余内容可删除？
- [ ] 每个要点是否足够精简？
- [ ] 是否用最少的词表达了最多的意思？

## 二、格式检查

### Marp语法
- [ ] YAML frontmatter 是否完整？
- [ ] `---` 分页符是否正确？
- [ ] Header/Footer 是否设置？
- [ ] 缩进是否一致（2空格或4空格）？

### Markdown格式
- [ ] 标题层级是否正确（h1→h2→h3）？
- [ ] 列表格式是否统一（无序/有序）？
- [ ] 粗体和引用是否正确使用？
- [ ] 代码块语言是否标注？

## 三、密度检查（重点）

### 文字数量
| 检查项 | 标准 | 状态 |
|--------|------|------|
| 每页文字行数 | ≤ 8行 | |
| 每行字符数 | ≤ 40字符 | |
| 列表项数量 | ≤ 6项 | |
| 段落长度 | ≤ 3行 | |

### 内容密度分级

**绿色（通过）**：
- 内容清晰，留白充足
- 视觉焦点明确
- 阅读轻松

**黄色（警告）**：
- 内容略多但可接受
- 建议优化但非必须
- 边界案例

**红色（必须修改）**：
- 文字明显过多
- 可能溢出页面
- 必须拆分或精简

## 四、视觉检查

### 层次感
- [ ] 标题是否突出？
- [ ] 重点是否用粗体标记？
- [ ] 引用是否使用blockquote？
- [ ] 信息层次是否清晰？

### 平衡感
- [ ] 内容是否居中或合理分布？
- [ ] 上下左右是否有足够边距？
- [ ] 视觉重心是否稳定？

### 一致性
- [ ] 字体和样式是否统一？
- [ ] 列表符号是否一致？
- [ ] 页面风格是否协调？

## 五、PNG检查专项

### 溢出检查
- [ ] 文字是否被截断？
- [ ] 内容是否超出可视区域？
- [ ] 页脚是否正常显示？

### 渲染检查
- [ ] 中文是否正确显示？
- [ ] 特殊字符是否渲染正确？
- [ ] 代码高亮是否正常？

### 布局检查
- [ ] 各元素位置是否正确？
- [ ] 间距是否合理？
- [ ] 整体布局是否美观？

## 六、问题处理指南

### 溢出问题
```
问题：文字超出页面
解决：
1. 拆分为多页
2. 精简文字内容
3. 使用更简洁的表达
```

### 密度过高
```
问题：单页内容过多
解决：
1. 删除次要信息
2. 将详情移至附录
3. 使用子页面展开
```

### 层次不清
```
问题：信息平铺无重点
解决：
1. 使用粗体标记重点
2. 调整标题层级
3. 使用引用块突出关键内容
```

## 七、评分标准

| 评分 | 标准 |
|------|------|
| A (优秀) | 内容精准、格式规范、密度适中、视觉美观 |
| B (良好) | 基本达标，有小问题但不影响展示 |
| C (及格) | 能用但需优化，存在明显可改进空间 |
| D (不及格) | 有严重问题必须修改（如溢出、逻辑断裂） |

## 八、快速检查流程

1. **5秒扫描**：每页快速浏览，感受密度
2. **内容验证**：核对是否覆盖所有要点
3. **格式检查**：确认Markdown语法正确
4. **PNG验证**：转换后逐页检查渲染结果
5. **整体评估**：给出综合评分和修改建议
